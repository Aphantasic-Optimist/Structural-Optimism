## Zenodo DOI

### What is Zenodo?

**Zenodo** is a research data repository operated by CERN that provides:
- Permanent DOI assignment
- Long-term archival (50+ years guaranteed)
- Open access hosting
- Version control
- Academic citability
- Metadata preservation

### Why We Use Zenodo

**Academic standards:**
- DOI is standard for academic citation
- Permanent identifier (won't break)
- Archival preservation (won't disappear)
- Version tracking (can cite specific versions)
- Metadata for discovery (findable)

**Open access:**
- Free to access
- Free to archive
- No paywalls
- No institutional barriers
- Globally accessible

**Integration:**
- Automatic GitHub integration
- Releases automatically archived
- DOI automatically assigned
- Metadata automatically extracted

### Accessing Archived Versions

**Each release is archived:**
- Version 1.0: `10.5281/zenodo.XXXXXXX`
- Version 1.1: `10.5281/zenodo.XXXXXXY`
- Version 2.0: `10.5281/zenodo.XXXXXXXZ`

**Concept DOI (always latest):** `10.5281/zenodo.XXXXXXX`

**Use concept DOI for general citations, specific version DOI for precise references.**

## Peer Review

### Anonymous Peer Review Process

**We use Anonymous Github for double-blind peer review:**

**What is Anonymous Github?**
- Tool for anonymous peer review
- Hides author identities from reviewers
- Hides reviewer identities from authors
- Maintains academic standards
- Preserves anonymity throughout

**Why anonymous peer review?**
- Reduces bias (focus on ideas, not identity)
- Protects junior researchers
- Enables honest criticism
- Prevents conflicts of interest
- Maintains academic rigor

### How to Request Peer Review

**For major contributions:**

1. **Submit your contribution** via pull request
2. **Request peer review** in PR description: `@structural-optimism-bot request-review`
3. **We set up Anonymous Github** - Creates anonymous repository
4. **Reviewers are assigned** - Experts in relevant field
5. **Review process begins** - Double-blind feedback
6. **You respond anonymously** - Address reviewer concerns
7. **Iterate until acceptance** - Revise based on feedback
8. **Merge when approved** - Contribution integrated

**See [Peer Review Guide](peer-review.md) for detailed process.**

### Peer Review Standards

**Reviewers evaluate:**
- Evidence quality (are sources reliable?)
- Reasoning validity (is logic sound?)
- Clarity (is it understandable?)
- Originality (is it novel?)
- Significance (does it matter?)
- Rigor (is it thorough?)

**Acceptance criteria:**
- Evidence is peer-reviewed and replicated
- Reasoning is logically valid
- Claims are falsifiable
- Contribution is significant
- Writing is clear
- Standards are met

### Becoming a Reviewer

**We need reviewers with expertise in:**
- Philosophy (especially philosophy of mind)
- Neuroscience (consciousness research)
- Psychology (social psychology, positive psychology)
- Physics (quantum mechanics, consciousness theories)
- Anthropology (cross-cultural studies)
- Statistics (meta-analysis, research methods)

**To volunteer as reviewer:**
1. Open issue: "Volunteer as reviewer"
2. Describe expertise
3. Provide credentials (optional, can be anonymous)
4. We'll add you to reviewer pool

**Reviewer commitment:**
- Review 2-4 contributions per year
- Provide constructive feedback
- Maintain anonymity (if desired)
- Follow review guidelines
- Meet deadlines (2-4 weeks per review)

## Academic Contributions

### Types of Academic Contributions

**We welcome:**

**1. Empirical evidence**
- New studies testing framework
- Meta-analyses of existing research
- Systematic reviews
- Replication studies

**2. Theoretical contributions**
- Extensions of framework
- Integration with other theories
- Resolution of contradictions
- Novel applications

**3. Critical analysis**
- Identification of weaknesses
- Challenges to claims
- Alternative explanations
- Boundary conditions

**4. Methodological improvements**
- Better measurement approaches
- Improved study designs
- Statistical innovations
- Replication protocols

### Contribution Process

**For academic contributions:**

1. **Review existing framework** - Understand current state
2. **Identify contribution** - What are you adding?
3. **Follow academic standards** - Peer-reviewed, rigorous
4. **Submit via GitHub** - Pull request with documentation
5. **Request peer review** - Anonymous if desired
6. **Respond to feedback** - Revise based on reviews
7. **Integration** - Accepted contributions integrated
8. **Credit** - You receive appropriate attribution

**See [Contributing Guide](../../CONTRIBUTING.md) for details.**

### Academic Standards

**All academic contributions must:**
- Be based on peer-reviewed research
- Use appropriate methodology
- Report results honestly
- Acknowledge limitations
- Cite sources properly
- Follow ethical guidelines
- Be reproducible

**We reject:**
- Fabricated data
- Plagiarism
- P-hacking
- HARKing (hypothesizing after results known)
- Selective reporting
- Conflicts of interest (undisclosed)

**Q: Is this peer-reviewed?**  
A: The framework undergoes community peer review. Individual claims cite peer-reviewed sources.

**Q: Can I request formal peer review?**
